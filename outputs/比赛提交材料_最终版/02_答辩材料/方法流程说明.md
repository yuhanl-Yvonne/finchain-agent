# 方法流程说明

## 一、总体思路
本项目围绕“低空经济白名单企业识别”构建了一条从数据整合到模型推荐的完整链路，具体包括：

1. 多源企业数据整合
2. Neo4j 知识图谱构建
3. 企业特征矩阵构建
4. GraphSAGE 图嵌入训练
5. XGBoost 融合排序
6. Neo4j 图谱核验与人工复核

## 二、数据层
- 企业基础数据：企业主表、产业链整合清单、企业画像数据
- 图谱关系数据：企业-产业链、企业-区域、企业-风险事件、企业-专利
- 标签数据：`企业主表.csv` 中的 `样本标记`

## 三、图谱层
- 节点类型：
  - `Company`
  - `ChainPosition`
  - `Region`
  - `RiskCase`
  - `Patent`
- 核心关系：
  - `(:Company)-[:BELONGS_TO_CHAIN]->(:ChainPosition)`
  - `(:Company)-[:IN_REGION]->(:Region)`
  - `(:Company)-[:HAS_RISK_EVENT]->(:RiskCase)`
  - `(:Company)-[:APPLIED_FOR]->(:Patent)`

## 四、特征层
- 结构化特征来自企业画像矩阵，包括：
  - 企业年龄
  - 注册资本
  - 参保人数
  - 地址与经营范围长度特征
  - 年鉴宏观环境特征
  - 专利面板统计特征
- 在建模前已删除明显标签泄漏字段，如样本标记、直接判定标签、产业链人工结论字段等。

## 五、图模型层
- 以 `Company` 为唯一预测节点构建同构图。
- 连边规则：
  - 同城市建立边
  - 同产业链环节建立边
- 使用两层 GraphSAGE 模型学习企业图嵌入。
- 输出每家企业的：
  - `graphsage_score`
  - `graphsage_embedding`

## 六、融合模型层
- 将 GraphSAGE embedding 与结构化特征拼接后输入 XGBoost。
- 输出：
  - `xgb_fusion_score`
  - `xgb_fusion_pred_label`
  - `white_list_level`
- 当前融合模型测试集指标：
  - F1 = 0.9231
  - AUC = 1.0000

## 七、结果交付层
- 结果文件：
  - `XGBoost融合白名单推荐表.xlsx`
  - `最终白名单人工核验表.xlsx`
  - `neo4j验图查询清单.md`
- 图谱回写字段：
  - `xgb_fusion_score`
  - `xgb_fusion_pred_label`
  - `white_list_level`
  - `graphsage_score`

## 八、项目亮点
- 不是只做表格分类，而是先构图再建模。
- 兼顾“头部企业识别”与“补链企业发现”。
- 实现了“数据整合 - 图谱建模 - 图神经网络 - 融合排序 - 人工核验”的闭环流程。
